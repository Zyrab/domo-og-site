# @zyrab/domo-og

სწრაფი, მრავალპლატფორმიანი, კონფიგურაციაზე დაფუძნებული Open Graph (OG) სურათების გენერატორი Node.js-ისთვის.

## შესავალი

`@zyrab/domo-og` არის თანამედროვე ხელსაწყო თქვენი ვებგვერდების, ბლოგებისა და აპლიკაციებისთვის მაღალი ხარისხის Open Graph სურათების დინამიურად დასაგენერირებლად.

ისტორიულად, SVG-ების გენერირება და მათი PNG-ად კონვერტაცია მოითხოვდა ოპერაციულ სისტემაზე დამოკიდებულ ბინარულ ფაილებს ან მძიმე ბრაუზერებს, როგორიცაა Puppeteer. `@zyrab/domo-og` ამ პრობლემას ჭრის WebAssembly-ის ([@resvg/resvg-wasm](https://www.npmjs.com/package/@resvg/resvg-wasm)) გამოყენებით. ის სრულყოფილად მუშაობს Windows, macOS და Linux სისტემებზე რთული დამოკიდებულებების გარეშე.

გარდა ამისა, ნაცვლად იმისა, რომ იმუშაოთ რთულ SVG სტრიქონებთან, თქვენ აპროექტებთ თქვენს სურათებს სუფთა, წაკითხვადი JSON კონფიგურაციის ობიექტების გამოყენებით.

## რატომ Domo-OG?

როდესაც საქმე ეხება Open Graph სურათების ავტომატურ გენერირებას, სიჩქარეს დიდი მნიშვნელობა აქვს — განსაკუთრებით სტატიკური საიტის გენერატორებისთვის (SSG), რომლებსაც შესაძლოა ათასობით გვერდის აწყობა დასჭირდეთ.

*   **Puppeteer / Playwright:** აშვებს მთლიან ბრაუზერს მხოლოდ სქრინშოტის გადასაღებად. ცნობილია როგორც ნელი (1-3 წამი თითო სურათზე), რესურსების ინტენსიური და მეხსიერების გაჟონვისკენ მიდრეკილი.
*   **Satori:** ეფუძნება მორგებულ React/HTML/CSS პარსინგის ძრავას (Yoga) JavaScript-ში, რათა განლაგება SVG-ად გარდაქმნას. ძლიერად ტვირთავს პროცესორს.
*   **Domo-OG:** იყენებს სრულიად განსხვავებულ მიდგომას. მარტივი JSON კონფიგურაციის მიღებით, ჩვენ საერთოდ გვერდს ვუვლით HTML/CSS განლაგების ძრავებს და DOM პარსინგს. JSON მომენტალურად აისახება ბრტყელ SVG ტილოზე და დაუყოვნებლივ გადაეცემა მაღალი წარმადობის Rust ძრავას PNG-ის რენდერინგისთვის.

## ძირითადი მახასიათებლები

- ⚡ **WASM-ზე დაფუძნებული ძრავა:** მრავალპლატფორმიანი თავსებადობა ყოველგვარი დამატებითი კონფიგურაციის გარეშე. არ საჭიროებს ოპერაციულ სისტემაზე მორგებული ბინარული ფაილების ჩამოტვირთვას.
- 🛠️ **კონფიგურაციაზე დაფუძნებული შაბლონები:** განსაზღვრეთ განლაგება, ფონი, ტექსტი და სურათები მარტივი JavaScript ობიექტების მეშვეობით.
- 💉 **მონაცემების დინამიური ინექცია:** მარტივად შეცვალეთ ცვლადები, როგორიცაა `{{title}}` ან `{{author}}` გენერირების დროს.
- 📐 **ტექსტის ჭკვიანი გადატანა:** ავტომატურად ითვლის ტექსტის წყვეტებს და მრავალხაზიან პოზიციონირებას.
- 💾 **ჩაშენებული ქეშირება:** თავიდან აგაცილებთ ზედმეტ გენერირებას `og-cache.json` მანიფესტის მეშვეობით, რაც მნიშვნელოვნად აჩქარებს პროცესს.
- 🌐 **დისტანციური სურათების ჩამოტვირთვა:** უსაფრთხოდ ჩამოტვირთავს და ქეშირებას უკეთებს დისტანციურ ფონებსა და ელემენტებს ჩაშენებული ლიმიტებით.

## სწრაფი დაწყება

აქ მოცემულია ბლოგ პოსტისთვის OG სურათის გენერირების ძირითადი მაგალითი. სტანდარტულად, გენერატორი იყენებს ჩაშენებულ შაბლონს და ავტომატურად ამუშავებს პროექტის ფარდობით გზებს.

```javascript
import { generateOgImage } from "@zyrab/domo-og";

async function build() {
  const imagePath = await generateOgImage({
    slug: "my-first-post",           // Used for caching and file naming
    ogOutputPath: "public",          // Where the assets/og-images folder will be created
    title: "My Awesome First Post!", // Injects into {{title}} in the template
    type: "Blog Article"             // Injects into {{type}} if defined in template
  });

  console.log(`Generated successfully at: ${imagePath}`);
}

build();
```

## კონფიგურაციის API (შაბლონები)

Domo-OG-ის შაბლონები მხოლოდ JavaScript ობიექტებია. ეს გამორიცხავს SVG ტეგების ხელით მანიპულირების საჭიროებას.

### ძირითადი კონფიგურაცია

| პარამეტრი | ტიპი | სტანდარტული | აღწერა |
| :--- | :--- | :--- | :--- |
| `width` | `number` | `1200` | გენერირებული სურათის სიგანე. |
| `height` | `number` | `630` | გენერირებული სურათის სიმაღე. |
| `fontPath` | `string` | `null` | გზა ან URL მორგებული .ttf, .otf, ან .woff ფონტისთვის. |
| `background` | `object` | `null` | ფონის ფენის კონფიგურაცია. |
| `elements` | `array[]` | `[]` | ელემენტების ობიექტების მასივი (ტექსტი ან სურათები) ზემოდან დასარენდერებლად. |

### ფონის ობიექტი

| პარამეტრი | ტიპი | აღწერა |
| :--- | :--- | :--- |
| `type` | `"color"\|"image"` | განსაზღვრავს ფონის ტიპს. |
| `value` | `string` | Hex/rgb ფერის კოდი (სავალდებულოა, თუ ტიპი არის "color"). |
| `src` | `string` | გზა ან URL ფონის სურათისთვის (სავალდებულოა, თუ ტიპი არის "image"). |

## Elements მასივის ობიექტები

ელემენტები რენდერდება თანმიმდევრულად (მასივის ბოლო ელემენტი რენდერდება ყველაზე ზემოთ).

### ტექსტის ელემენტი

| პარამეტრი | ტიპი | აღწერა |
| :--- | :--- | :--- |
| `type` | `"text"` | განსაზღვრავს ელემენტის ტიპს. |
| `content` | `string` | საჩვენებელი ტექსტი. მხარს უჭერს ცვლადებს, მაგ: `{{title}}`. |
| `width` | `number` | ტექსტის ელემენტის სიგანე. |
| `maxLength` | `number` | ტექსტის ელემენტის მაქსიმალური სიგრძე გადატანამდე. |
| `horizontalAlign` | `"left"\|"center"\|"right"` | ჰორიზონტალური სწორება padding-ზე/სიგანეზე დაყრდნობით. |
| `verticalAlign` | `"top"\|"middle"\|"bottom"` | ვერტიკალური სწორება padding-ზე/სიმაღლეზე დაყრდნობით. |
| `fontSize` | `number` | ფონტის ზომა პიქსელებში (სტანდარტული: 32). |
| `color` | `string` | Hex/rgb ტექსტის ფერი. |
| `backgroundColor` | `string` | ამატებს გამოყოფილ ფონს ტექსტის უკან. |
| `borderRadius` | `number` | ტექსტის ფონის მომრგვალება. |
| `padding` | `number` | მანძილი ტილოს კიდეებიდან. |
| `bgPadding` | `number` | დაშორება ტექსტსა და მის backgroundColor ფონს შორის. |

### სურათის ელემენტი

| პარამეტრი | ტიპი | აღწერა |
| :--- | :--- | :--- |
| `type` | `"image"` | განსაზღვრავს ელემენტის ტიპს. |
| `src` | `string` | ლოკალური გზა ან დისტანციური URL სურათისთვის. |
| `width` | `number` | სურათის სიგანე (სტანდარტული: 100). |
| `height` | `number` | სურათის სიმაღლე (სტანდარტული: 100). |
| `horizontalAlign` | `"left"\|"center"\|"right"` | ჰორიზონტალური სწორება ტილოზე. |
| `verticalAlign` | `"top"\|"middle"\|"bottom"` | ვერტიკალური სწორება ტილოზე. |
| `padding` | `number` | მანძილი ტილოს კიდეებიდან. |

## გაფართოებული გამოყენება

### გაფართოებული შაბლონის მაგალითი

აქ მოცემულია გაფართოებული შაბლონი, რომელიც გვიჩვენებს დინამიურ მონაცემებს, დისტანციურ სურათებს და მორგებულ სტილს:

```javascript
const advancedTemplate = {
  width: 1200,
  height: 630,
  background: {
    type: "color",
    value: "#1a1a1a",
  },
  elements: [
    {
      type: "image",
      src: "https://example.com/mock-logo.png", // დისტანციური სურათის მაგალითი
      horizontalAlign: "left",
      verticalAlign: "top",
      padding: 40,
      width: 80,
      height: 80,
    },
    {
      type: "text",
      content: "{{category}}", // დინამიური ცვლადი
      horizontalAlign: "left",
      verticalAlign: "top",
      padding: 140, // ჩამოწეულია ლოგოს ქვემოთ
      fontSize: 24,
      color: "#000000",
      backgroundColor: "#E9FA00",
      bgPadding: 8,
      borderRadius: 4,
    },
    {
      type: "text",
      content: "{{title}}", // დინამიური ცვლადი
      horizontalAlign: "center",
      verticalAlign: "middle",
      fontSize: 64,
      color: "#FFFFFF",
      maxLength: 20 // აიძულებს ხაზის გადატანას ~20 სიმბოლოს შემდეგ
    }
  ]
};
```

## ფონტები და ალტერნატივები

WASM რენდერინგის ძრავას სჭირდება ვალიდური `.ttf`, `.otf`, ან `.woff` ფონტის ფაილი ტექსტის დასარენდერებლად. `@zyrab/domo-og` ამას გონივრულად აკეთებს:

1.  **მითითებული URL/გზა:** თუ თქვენს კონფიგურაციაში მიუთითებთ `fontPath`-ს, ის ჩამოიტვირთება/ჩაიტვირთება.
2.  **ლოკალური ფაილები:** ის ავტომატურად დაასკანერებს `${ogOutputPath}/assets/fonts/` ნებისმიერი ვალიდური ფონტისთვის.
3.  **შიდა ალტერნატივა:** თუ არცერთი არ მოიძებნა, ის უსაფრთხოდ გადაერთვება პაკეტში ჩაშენებულ სტანდარტულ ფონტზე (`fonts/default.ttf`).

## დისტანციური სურათების უსაფრთხოება და ლიმიტები

რათა დაიცვათ თქვენი გენერირების პროცესები მეხსიერების გაჟონვისგან და შეფერხებებისგან:

-   დისტანციურ სურათებს აქვთ მკაცრი **20MB** ზომის ლიმიტი.
-   პაკეტი იყენებს მორგებულ User-Agent-ს: `Domo-OG-Builder/1.0`. დარწმუნდით, რომ დაამატებთ ამ User-Agent-ს თეთრ სიაში, თუ თქვენი სერვერი ბლოკავს უცნობ აგენტებს.

## ძირითადი API რეფერენსი

### `generateOgImage(options)`

პაკეტის ძირითადი ფუნქცია. ის აწყობს SVG-ს, არენდერებს PNG-ად და მართავს ქეშს.

**პარამეტრები:**

*   `slug` (სტრიქონი, სავალდებულო): სურათის უნიკალური იდენტიფიკატორი (ხშირად URL slug).
*   `ogOutputPath` (სტრიქონი, სავალდებულო): გამომავალი დირექტორია (მაგ: "public"). სურათები ინახება `${ogOutputPath}/assets/og-images/` დირექტორიაში.
*   `template` (ობიექტი, არასავალდებულო): თქვენი კონფიგურაციის შაბლონი. გამოტოვების შემთხვევაში გამოიყენება სტანდარტული.
*   `routeKey` (სტრიქონი, არასავალდებულო): ალტერნატიული გასაღები ჰეშირებისთვის/ქეშირებისთვის.
*   `...flatParams`: ნებისმიერი სხვა გასაღები იქნება ინტეგრირებული (მაგ: `title: "Hello"` ჩაანაცვლებს `{{title}}`-ს).

**აბრუნებს:** `Promise<String>`: გენერირებული PNG-ის ფარდობითი გზა.

### `getDefaultConfig()`

აბრუნებს სუფთა, თანამედროვე, მუქი თემის სტანდარტულ კონფიგურაციის ობიექტს.

### `flushManifestImmediately()`

აიძულებს ქეშირების მანიფესტს (`og-cache.json`) დაუყოვნებლივ ჩაიწეროს დისკზე.
